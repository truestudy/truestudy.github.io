2. tabbed document interface (tdi) memberi peluang pada beberapa dokumen ditampilkan pada...​


Tabbed document interface (TDI) memungkinkan beberapa dokumen untuk ditampilkan pada satu jendela dan menggunakan tab sebagai alat navigasi. TDI biasanya kita temukan di perangkat lunak, salah satunya adalah web browser.

Pembahasan

Tabs adalah salah satu elemen penting dalam UI (User Interface) sebuah browser. Awalnya, browser tidak memiliki tabs seperti yang kita kenal sekarang. Hal ini disebabkan karena waktu itu kebutuhan pengguna akan web browser belum sekompleks saat ini.

Elemen tabs kemudian muncul untuk memudahkan pengguna dalam mengakses banyak halaman website sekaligus. Hingga kini, tabs sudah menjadi bagian yang wajib ada dalam setiap web browser.

Untuk membuka tab baru di browser, kita bisa menggunakan shortcut CTRL + T pada keyboard. Berikut ini adalah jenis dan perkembangan interface tabs:

Single Document Interface (SDI)

SDI adalah tampilan dimana aplikasi tersebut hanya bisa menampilkan satu dokumen saja dalam satu jendela. Hingga saat ini, masih ada beberapa aplikasi yang menggunakan tampilan Single Document Interface, contohnya: Microsoft Paint, Windows Explorer, Notepad.

Multiple Document Interface (MDI)

MDI adalah interface dimana dalam 1 window, terdapat beberapa sub-window yang bisa digunakan untuk membuka dokumen lain. Nantinya sub-window ini dapat disusun agar tak saling bertumpuk.

Tabbed Document Interface (TDI)

Sebenarnya, TDI adalah pengembangan dari MDI. Konsep dari kedua interface ini sama, yakni memungkinkan aplikasi bisa membuka banyak dokumen dalam satu jendela.

Bedanya, dalam TDI navigasinya menggunakan sistem tabs yang lebih praktis. User dapat berpindah-pindah tampilan dokumen, tapi mereka bisa dengan mudah fokus ke 1 dokumen saja dalam satu waktu.

28. bagaimana cara pembuatan user interface yang dapat menyisipkan sebuah document atau gambar?,pada gambar di atas​


Jawaban:

Dengan menggunakan CSS atau framework seperti Botstrap, Tailwind dan lain sebagainya. tetapi supaya cara kerja beraturan dalam membuat UI, dibuat dahulu prototype nya menggunakan software seperti Figma, adobe xd dll supaya tau batasan kerja, color pallete, atau element lain yg akan di buat

Jawaban:

Kamu harus buat code-nya di dalam form.

<form>

//code kamu

</form>

__

Kalau buat kotak inputan, biasanya kan pakai input type-nya yang jenis text.

< input type = "text" >

Nah, untuk menyisipkan file, pakai yang jenis file.

< input type = "file" >

Kita juga pakai atribut accept = " " untuk memilih jenis file yang diinginkan.

<form>

<label for ="myfile" > Foto : </label>

<input type = "file" id "myfile" name= "myfile" accept ="image/*,.docx" multiple>

</form>

____

Atribut accept ="image/*,.docx" untuk menampilkan seluruh gambar dan dokumen dengan format .docx

Kamu bisa menampilkan format dokumen yang kamu inginkan. Misal jenis .doc, .xlsx, .pdf . Jangan lupa pakai tanda titik ( . ) sebelum jenis format  file dan tanda koma ( , ) sebagai pemisah jenis format.

accept ="image/*, .docx, .xlsx, .pdf"

Untuk menampilkan seluruh format gambar pakai tanda bintang *.

accept = "image/*"

Untuk menampilkan format gambar tertentu, tulis format gambar yang diinginkan. Misal format .gif

accept = "image/.gif"

Menampilkan lebih dari satu gambar tertentu dengan format berbeda langsung saja.

accept =".gif, .jpeg, .png"

Misal, hanya ingin menampilkan gambar dan dokumen (jpg, png, docx, doc, xls)

accept =".jpg, .png, .docx, .doc, .xls"

__

Atribut multiple berfungsi untuk bisa memilih seluruh file sekaligus.

Jika kamu hanya ingin memilih satu dokumen atau satu gambar saja, tidak perlu ditulis atribut multiple -nya dan jadinya seperti ini:

<input type = "file" id "myfile" name= "myfile" accept ="image/*,.docx">
