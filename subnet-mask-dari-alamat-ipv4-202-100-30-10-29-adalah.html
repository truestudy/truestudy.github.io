5. Berapa subnet mask dari alamat IP 181.180.10.7/20


Jawaban:

subnet mask dari alamat IP 181.180.10.7/20 adalah 255.255.240.0

Penjelasan:

Untuk mengetahui subnet mask dari alamat IP 181.180.10.7/20, kita perlu mengonversi prefix length (20 bit) menjadi notasi desimal dari subnet mask.

Kita tahu bahwa subnet mask adalah serangkaian bit 1 diikuti oleh serangkaian bit 0. Dalam kasus ini, kita memiliki 20 bit 1 diikuti oleh 12 bit 0. Oleh karena itu, subnet mask dapat ditulis sebagai 255.255.240.0, di mana 240 dalam notasi biner adalah 11110000.

17. Tentukan subnet,alamat bradcast,dan jangkauan host yang valid untuk IP berikuta. Alamat node :192.168.10.33 dan Subnet mask 255.255.255.224b. Alamat node :192.168.10.33 dan Subnet mask 255.255.255.240​


a.

menentukan class

192.168.10.33 ⇒ merupakan class C

kemudian kita menentukan nilai CIDR dari subnet mask

subnet mask 255.255.255.224 ⇒ /27

jadi, network address 192.168.10.33/27

Hitung jumlah subnet

Jumlah Subnet = [tex]2^{x}[/tex], dimana x adalah banyaknya binari 1 pada oktet terakhir subnet mask.

menentukan nilai x ubah subnet mask terlebih dulu kebiner kemudian hitung jumlah 1 pada ip class c255.255.255.224 ⇒ 11111111.11111111.11111111.11100000karena class C kita menghitung angka 1 setelah deret ke 24, sehingga berjumlah tiga angka 1 (lihat angka bergaris bawah)11111111.11111111.11111111.11100000

jadi jumlah subnet [tex]2^{3}[/tex] = 8

Hitung Jumlah Host per Subnet

Jumlah Host per Subnet = [tex]2^{y}[/tex] – 2, dimana y adalah adalah kebalikan dari x yaitu banyaknya binari 0 pada oktet terakhir subnet.

menentukan nilai y sama dengan menentukan nilai x pada jumlah subnet diatas , tapi bedanya yang dihitung angka 0 bukan 1 pada deret biner. (lihat angka bergaris bawah)11111111.11111111.11111111.11100000

jadi jumlah subnet [tex]2^{5}[/tex] = 32

Menghitung Blok Subnet

Blok Subnet = 256 – (nilai oktet terakhir subnet mask).

menentukan nilai oktet terakhir subnet mask.mari kita lihat lagi subnet masknya255.255.255.224sehingga bisa kita ambil nilai oktet terakhirnya (lihat angka bergaris bawah)255.255.255.224

jadi Blok Subnet = 256 – 224 = 32, Subnet berikutnya adalah 32 + 32 = 64,  64 + 32 = 96 , 96 + 32 = 128, 128 + 32 = 160, 160 + 32 = 192, 192+ 32 = 224

host dan broadcast

sehingga kita dapat menyusun tabel host dan broadcast. (lihat pada gambar)
