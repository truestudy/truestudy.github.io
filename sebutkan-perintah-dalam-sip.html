1. jelaskan 7 perintah pada SIP​


Berikut ini beberapa perintah pada SIP​, yang mana diantaranya ialah:

INVITE, yang mana dia sendiri akan mengindikasikan jika user atau layanan sedang diundang untuk bergabung dengan sesi.ACKNOWLEDGEMENT atau ACK, yang mana dia sendiri akan mengkonfirmasi jika client sudah menerima respon terakhir untuk permintaan INVITE, dan dia hanya digunakan pada permintaan REQUEST.OPTION, yang mana dia sendiri akan digunakan untuk mempertanyakan server tentang kemampuannya.BYE, yang mana dia sendiri akan dikirim oleh client user agent untuk mengindikasikan server jika percakapan akan segera dimatikan.CANCEL, yang mana dia sendiri untuk membatalkan permintaan yang sedang menunggu keputusan REGISTER, yang mana dia sendiri akan digunakan oleh client untuk mendaftarkan informasi kontak Respon pesan berisi kode status dan informasi tentang kondisi permintaan.

Pembahasan

SIP sendiri pada dasarnya dapat kita maknai sebagai suatu singkatan dari Session Initiation Protocol yang dimana dia ialah protocol pensinyalan pada layer aplikasi yang berfungsi untuk membangun, memodifikasi, dan mengakhiri suatu sesi multimedia yang melibatkan satu user atau lebih. Selain itu Sesi ini sendiri ialah pertukaran data antar user yang meliputi suara,video,dan text, yang mana SIP tidak menyediakan layanan secara langsung tapi menyediakan layanan yang dapat digunakan oleh protokol aplikasi lainnya untuk memberikan layanan yang lebih lengkap.

2. Jelaskan 7 perintah pada SIP


SIP adalah singkatan dari Session Intial Protocol. SIP sendiri adalah protokol pensinyalan dan lapisan kontrol atau aplikasi. Fungsi SIP adalah untuk membangun dan mengakhiri sesi multimedia. Berikut di bawha ini adalah 7 perintah yang ada pada SIP:

INVITE adalah perintah untuk mengundang pengguna ke panggilan.ACK atau acknowledgment adalah perintah yang digunakan untuk memudahkan pertukaran pesan melalui perintah INVITE.BYE adalah perintah untuk memutuskan koneksi antar pengguna.CANCEL adalah perintah untuk berhenti meminta atau mencari pengguna. Perintah ini digunakan saat klien mengirimkan perintah INVITE dan mengubah keputusannya untuk memanggil penerima.OPSI adalah perintah untuk mengumpulkan informasi tentang kemampuan server.REGISTER adalah perintah untuk mendaftarkan lokasi pengguna saat ini.INFO adalah perintah yang digunakan di tengah sesi pensinyalan.

Pembahasan

SIP adalah singkatan dari Session Intial Protocol. SIP sendiri adalah protokol pensinyalan dan lapisan kontrol atau aplikasi. Fungsi SIP adalah untuk membangun dan mengakhiri sesi multimedia.

SIP sendiri pada dasarnya dapat diartikan sebagai singkatan dari Session Initiation Protocol, sebuah protokol lapisan aplikasi yang digunakan untuk membuat, memodifikasi, dan mengakhiri sesi multimedia yang melibatkan satu atau lebih pengguna. langsung dari SIP, melainkan menyediakan layanan suara, video, dan teks yang dapat digunakan oleh protokol aplikasi lain untuk memberikan layanan yang lebih lengkap.

22. Perbedaan sip request dan sip server


SIP server merupakan aplikasi yang bertujuan untuk menerima. Sementara itu, SIP request merupakan aplikasi yang bertujuan untuk menjawab.

Pembahasan

SIP merupakan singkatan dari Session Initiation Protocol. SIP sendiri merupakan suatu persinyalan protokol dan pengontrolan lapisan atau layer aplikasi. Fungsi dari SIP adalah untuk membangun dan mengakhiri sebuah sesi multimedia.

Sesi multimedia merupakan sebutan bagi sesi di mana pengguna suatu aplikasi saling bertukar data. Data tersebut dapat berupa teks, suara, maupun video. Di situ, tugas SIP adalah untuk menyediakan dasar bagi protokol aplikasi lain.

Berikut ini adalah fungsi 4 server SIP secara singkat:

Proxy server, yaitu server yang berfungsi sebagai server dan client. Server ini juga melakukan routing, memastikan request untuk pihak yang berizin, dan membuat kebijakan.Redirect server, yaitu server yang berfungsi untuk menyelesaikan tugas request yang diberikan pengguna aplikasi.Registrar server, yaitu server yang berfungsi unutk menerima dan melakukan proses pada pemberitahuan pendaftaran dengan cara mengizinkan akses lokasi keberadaan suatu endpoint.Location server, yaitu server yang berfungsi untuk menyediakan layanan pada database abstrak dengan tujuan menerjemahkan keterangan pada domain jaringan.

Definisi dari SIP adalah suatu sesi tentang masa terjadi pertukaran data antar user yang meliputi data suara, data video, dan data text. Berikut ini adalah beberapa unsur dan penjelasan komponen standar pada SIP (Session Initiation Protocol):

Agen Pengguna (User Agent): Agen pengguna berada di setiap stasiun akhir SIP.Redirect Server: Alamat yang ditentukan selama inisialisasi. Server Proxy: Identitas perantara yang bertindak  sebagai server dan klien, membuat permintaan atas nama klien lain. Registrar atau Paniter: Server yang menerima permintaan pendaftaran dan menempatkan informasi yang diterima. Layanan Lokasi: Layanan lokasi yang digunakan oleh redirector SIP atau server proxy untuk mendapatkan informasi tentang kemungkinan target.

Pembahasan

Definisi dari SIP adalah suatu sesi tentang masa terjadi pertukaran data antar user yang meliputi data suara, data video, dan data text. SIP tidak menyediakan layanan secara langsung, tetapi menyediakan layanan yang dapat digunakan oleh protokol aplikasi lain untuk menyediakan serangkaian layanan yang lebih lengkap kepada pengguna. Misalnya, ia menggunakan RTP (Real Time Transport Protocol) untuk transmisi data waktu nyata dan SDP (Session Description Protocol). Sesi menjelaskan komunikasi multimedia  dengan PSTN (Public Switch Telephone Network) menggunakan MEGACO (Media Gateway Control Protocol). Oleh karena itu, fungsionalitas dasar dan operasi  SIP tidak bergantung pada protokol ini. SIP juga independen dari protokol lapisan transport yang digunakan.

Perkembangan komunikasi multimedia dengan menggunakan SIP terjadi dalam beberapa tahap:

Lokasi Pengguna: Menentukan lokasi pengguna yang berkomunikasi dengan kita. Ketersediaan Pengguna: Menentukan tingkat keinginan pihak yang dipanggil untuk berpartisipasi dalam komunikasi yang diinginkan. Kegunaan: Tentukan media dan parameter terkait media yang  digunakan untuk komunikasi. Pengaturan sesi: "Dering", pengaturan koneksi antara penelepon dan pihak yang dipanggil. Manajemen sesi: termasuk transfer, modifikasi, dan penghentian sesi.
